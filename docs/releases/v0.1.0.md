# ShipStack v0.1.0 Release Notes

Status: release candidate. Publish this release after a real Cloudflare deploy
pass and remote npm publish workflow dry-run.

## What Is Included

- TanStack Start base app for Cloudflare Workers
- TanStack Router file routes
- Tailwind CSS with minimal reusable UI primitives
- `/`, `/health`, `/dashboard`, `/api/health`, and `/api/v1/me`
- restrictive trusted-origin CORS handling for API routes
- D1 + Drizzle module
- Better Auth email/password module
- protected dashboard and account routes
- generated app `lint`, `test`, `typecheck`, `build`, and `deploy` scripts
- generated app `deploy:dry-run` script
- `create-shipstack`
- `shipstack doctor`
- `shipstack add database`
- `shipstack add auth`
- `shipstack add billing`
- `shipstack add storage`
- `shipstack add api-keys`
- `shipstack add openapi`
- `shipstack add api-rate-limit`
- Stripe billing module with checkout, portal, webhook-confirmed subscription state, and entitlement status
- R2 storage module with session-authenticated `/api/v1/files`
- API keys recipe with hashed key storage, session-managed key lifecycle routes, and bearer auth support for `/api/v1/me`
- OpenAPI recipe with generated OpenAPI 3.1 docs, `public/openapi.json`, and `/api/openapi`
- API rate limit recipe with route-level fixed-window helpers and `RATE_LIMITED` JSON errors
- GitHub Actions verification and manual deploy workflows
- optional Cloudflare temporary deploy smoke
- English and Chinese documentation

## Supported Runtime

- Node.js 22 or newer is the supported target for repository and generated-app tooling.
- Cloudflare Workers is the supported deployment runtime.
- `nodejs_compat` is enabled in `wrangler.jsonc`.

## Supported Package Manager

- pnpm 10 is the supported package manager.
- The repository is configured with `packageManager: pnpm@10.33.0`.
- CI currently installs pnpm `10.33.0`.

## Required Cloudflare Setup

For the base app:

1. Log in with Wrangler.
2. Run `pnpm run deploy`.
3. Verify `/health` and `/api/health` on the deployed Worker URL.

For D1:

1. Run `pnpm db:cf:create`.
2. Copy the returned `database_id` into `wrangler.jsonc`.
3. Run `pnpm db:generate`.
4. Run `pnpm db:cf:migrate:local`.
5. Run `pnpm db:cf:migrate:remote`.

For Better Auth:

1. Set `BETTER_AUTH_SECRET` with Wrangler secrets.
2. Set `BETTER_AUTH_URL` to the deployed Worker origin.
3. Deploy again.
4. Verify sign up, sign out, sign in, and protected dashboard access.

For Stripe billing:

1. Set `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `STRIPE_PRICE_ID`, `BILLING_SUCCESS_URL`, `BILLING_CANCEL_URL`, and `BILLING_PORTAL_RETURN_URL`.
2. Run `pnpm db:generate`.
3. Run `pnpm db:cf:migrate:local`.
4. Run `pnpm db:cf:migrate:remote`.
5. Configure Stripe webhooks for `/api/stripe/webhook`.

For R2 storage:

1. Run `wrangler r2 bucket create shipstack-files`.
2. Confirm the `FILES` binding exists in `wrangler.jsonc`.
3. Run `pnpm db:generate`.
4. Run `pnpm db:cf:migrate:local`.
5. Run `pnpm db:cf:migrate:remote`.

## Verification

Quick release gate audit:

```bash
pnpm release:audit
```

In this workspace the audit currently reports that local gates, remote CI
evidence, git remote configuration, and Wrangler authentication pass, while
final release is still blocked by pending real Cloudflare deploy evidence and
pending remote npm publish workflow dry-run evidence.

Local verification currently passes:

```bash
pnpm verify:release
```

The smoke suite verifies generated base, database, auth, billing, storage, API
keys, OpenAPI, and API rate limit apps, including D1 migrations, API routes,
trusted-origin CORS behavior, browser auth flow, Stripe webhook idempotency and
entitlement status, R2 file upload/list/delete flow, API key creation/revocation
and bearer auth, OpenAPI generation and `/api/openapi`, API rate limiting,
generated-app lint checks, and a Wrangler deploy dry-run for the base app.
Release verification also checks npm package tarball contents with
`pnpm pack:check`. That check installs the packed tarballs into a temporary
workspace, creates an app with `create-shipstack`, installs database, auth,
billing, storage, API keys, OpenAPI, and API rate limit modules with the packed
`shipstack` CLI, and runs `shipstack doctor`.

The repository includes a manual `Release npm Packages` GitHub Actions workflow
for publishing `@shipstack/core`, `@shipstack/cli`, and `create-shipstack` with
npm provenance. Maintainers can run `pnpm publish:dry-run` locally to check npm
accepts the packed tarballs before running the workflow. Run the workflow in
dry-run mode before a real publish.

Optional external deployment verification can be run with:

```bash
pnpm smoke:temporary-deploy
```

This uses Cloudflare's temporary account flow to upload a generated app and
check the deployed `/health`, `/api/health`, and `/api/v1/me` routes. It has
passed before in this workspace, but the latest attempt on 2026-06-24 failed
before upload with Wrangler `fetch failed` connectivity errors.

## Known Limitations

- A real-account Cloudflare deploy pass has not been recorded yet.
- SDK generation, teams, and admin features are out of scope for this release.
- Only Cloudflare Workers-first providers are supported in the MVP path: Workers,
  D1, R2, and Stripe.
- OAuth provider setup is intentionally optional and not part of the local first-run path.
- The CLI module system is intentionally small and currently supports `database`, `auth`, `billing`, `storage`, `api-keys`, `openapi`, and `api-rate-limit`.
