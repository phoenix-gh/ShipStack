# ShipStack v0.1.0 Release Notes

状态：release candidate。真实 Cloudflare 部署验证和远端 CI 确认通过后，再发布这个 release。

## 包含内容

- 面向 Cloudflare Workers 的 TanStack Start 基础应用
- TanStack Router file routes
- Tailwind CSS 和最小可复用 UI primitives
- `/`、`/health`、`/dashboard`、`/api/health` 和 `/api/v1/me`
- API routes 的默认收紧 trusted-origin CORS handling
- D1 + Drizzle 模块
- Better Auth email/password 模块
- 受保护 dashboard 和 account routes
- 生成应用的 `lint`、`test`、`typecheck`、`build` 和 `deploy` scripts
- 生成应用的 `deploy:dry-run` script
- `create-shipstack`
- `shipstack doctor`
- `shipstack add database`
- `shipstack add auth`
- GitHub Actions verification 和手动 deploy workflows
- 可选 Cloudflare 临时部署 smoke
- 英文和中文文档

## 支持的运行环境

- Node.js 22 或更新版本是仓库和生成应用 tooling 的支持目标。
- Cloudflare Workers 是支持的部署 runtime。
- `wrangler.jsonc` 已启用 `nodejs_compat`。

## 支持的包管理器

- pnpm 10 是支持的包管理器。
- 仓库配置了 `packageManager: pnpm@10.0.0`。
- CI 当前安装 pnpm `10.33.0`。

## 必需的 Cloudflare 设置

基础应用：

1. 用 Wrangler 登录。
2. 运行 `pnpm deploy`。
3. 在部署后的 Worker URL 上验证 `/health` 和 `/api/health`。

D1：

1. 运行 `pnpm db:cf:create`。
2. 把返回的 `database_id` 复制到 `wrangler.jsonc`。
3. 运行 `pnpm db:generate`。
4. 运行 `pnpm db:cf:migrate:local`。
5. 运行 `pnpm db:cf:migrate:remote`。

Better Auth：

1. 用 Wrangler secrets 设置 `BETTER_AUTH_SECRET`。
2. 把 `BETTER_AUTH_URL` 设置为部署后的 Worker origin。
3. 再次部署。
4. 验证注册、退出、登录和受保护 dashboard 访问。

完整 checklist 见 [部署验证](../DEPLOYMENT.md)。

## 验证情况

当前本地验证通过：

```bash
pnpm verify:release
```

Smoke suite 会验证生成的 base、database 和 auth 应用，包括 D1 migrations、API routes、trusted-origin CORS 行为、浏览器 auth flow、生成应用 lint checks，以及 base 应用的 Wrangler deploy dry-run。

可选外部部署验证也已通过：

```bash
pnpm smoke:temporary-deploy
```

它会使用 Cloudflare temporary account flow 上传生成应用，并检查已部署的
`/health`、`/api/health` 和 `/api/v1/me` routes。

## 已知限制

- 还没有记录真实账号下的 Cloudflare 部署验证。
- 当前 workspace 还没有确认远端 GitHub Actions 结果。
- Billing、storage、API keys、OpenAPI、SDK generation、teams 和 admin features 不在本 release 范围。
- MVP 路径只支持 Cloudflare Workers 和 D1。
- OAuth provider setup 是可选项，不属于本地首次运行路径。
- CLI module system 目前刻意保持很小，只支持 `database` 和 `auth`。
