# ShipStack v0.1.0 Release Notes

Status: release candidate. Publish this release after a real Cloudflare deploy
pass and remote CI confirmation.

## What Is Included

- TanStack Start base app for Cloudflare Workers
- TanStack Router file routes
- Tailwind CSS with minimal reusable UI primitives
- `/`, `/health`, `/dashboard`, `/api/health`, and `/api/v1/me`
- restrictive trusted-origin CORS handling for API routes
- D1 + Drizzle module
- Better Auth email/password module
- protected dashboard and account routes
- generated app `lint`, `test`, `typecheck`, `build`, and `deploy` scripts
- generated app `deploy:dry-run` script
- `create-shipstack`
- `shipstack doctor`
- `shipstack add database`
- `shipstack add auth`
- GitHub Actions verification and manual deploy workflows
- optional Cloudflare temporary deploy smoke
- English and Chinese documentation

## Supported Runtime

- Node.js 22 or newer is the supported target for repository and generated-app tooling.
- Cloudflare Workers is the supported deployment runtime.
- `nodejs_compat` is enabled in `wrangler.jsonc`.

## Supported Package Manager

- pnpm 10 is the supported package manager.
- The repository is configured with `packageManager: pnpm@10.0.0`.
- CI currently installs pnpm `10.33.0`.

## Required Cloudflare Setup

For the base app:

1. Log in with Wrangler.
2. Run `pnpm deploy`.
3. Verify `/health` and `/api/health` on the deployed Worker URL.

For D1:

1. Run `pnpm db:cf:create`.
2. Copy the returned `database_id` into `wrangler.jsonc`.
3. Run `pnpm db:generate`.
4. Run `pnpm db:cf:migrate:local`.
5. Run `pnpm db:cf:migrate:remote`.

For Better Auth:

1. Set `BETTER_AUTH_SECRET` with Wrangler secrets.
2. Set `BETTER_AUTH_URL` to the deployed Worker origin.
3. Deploy again.
4. Verify sign up, sign out, sign in, and protected dashboard access.

See [Deployment Verification](../DEPLOYMENT.md) for the full checklist.
See [Release Checklist](../RELEASE.md) for the maintainer release flow.

## Verification

Local verification currently passes:

```bash
pnpm verify:release
```

The smoke suite verifies generated base, database, and auth apps, including D1
migrations, API routes, trusted-origin CORS behavior, browser auth flow,
generated-app lint checks, and a Wrangler deploy dry-run for the base app.
Release verification also checks npm package tarball contents with
`pnpm pack:check`.

Optional external deployment verification also passes:

```bash
pnpm smoke:temporary-deploy
```

This uses Cloudflare's temporary account flow to upload a generated app and
check the deployed `/health`, `/api/health`, and `/api/v1/me` routes.

## Known Limitations

- A real-account Cloudflare deploy pass has not been recorded yet.
- Remote GitHub Actions has not been confirmed in this workspace.
- Billing, storage, API keys, OpenAPI, SDK generation, teams, and admin features are out of scope for this release.
- Only Cloudflare Workers and D1 are supported in the MVP path.
- OAuth provider setup is intentionally optional and not part of the local first-run path.
- The CLI module system is intentionally small and currently supports only `database` and `auth`.
