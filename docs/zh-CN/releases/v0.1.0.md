# ShipStack v0.1.0 Release Notes

状态：release candidate。真实 Cloudflare 部署验证和远端 CI 确认通过后，再发布这个 release。

## 包含内容

- 面向 Cloudflare Workers 的 TanStack Start 基础应用
- TanStack Router file routes
- Tailwind CSS 和最小可复用 UI primitives
- `/`、`/health`、`/dashboard`、`/api/health` 和 `/api/v1/me`
- API routes 的默认收紧 trusted-origin CORS handling
- D1 + Drizzle 模块
- Better Auth email/password 模块
- 受保护 dashboard 和 account routes
- 生成应用的 `lint`、`test`、`typecheck`、`build` 和 `deploy` scripts
- 生成应用的 `deploy:dry-run` script
- `create-shipstack`
- `shipstack doctor`
- `shipstack add database`
- `shipstack add auth`
- `shipstack add billing`
- `shipstack add storage`
- `shipstack add api-keys`
- `shipstack add openapi`
- `shipstack add api-rate-limit`
- Stripe billing 模块，包含 checkout、portal、webhook-confirmed subscription state 和 entitlement status
- R2 storage 模块和 session-authenticated `/api/v1/files`
- API keys recipe，包含 hashed key storage、session-managed key lifecycle routes，以及 `/api/v1/me` bearer auth support
- OpenAPI recipe，包含 generated OpenAPI 3.1 docs、`public/openapi.json` 和 `/api/openapi`
- API rate limit recipe，包含 route-level fixed-window helpers 和 `RATE_LIMITED` JSON errors
- GitHub Actions verification 和手动 deploy workflows
- 可选 Cloudflare 临时部署 smoke
- 英文和中文文档

## 支持的运行环境

- Node.js 22 或更新版本是仓库和生成应用 tooling 的支持目标。
- Cloudflare Workers 是支持的部署 runtime。
- `wrangler.jsonc` 已启用 `nodejs_compat`。

## 支持的包管理器

- pnpm 10 是支持的包管理器。
- 仓库配置了 `packageManager: pnpm@10.33.0`。
- CI 当前安装 pnpm `10.33.0`。

## 必需的 Cloudflare 设置

基础应用：

1. 用 Wrangler 登录。
2. 运行 `pnpm deploy`。
3. 在部署后的 Worker URL 上验证 `/health` 和 `/api/health`。

D1：

1. 运行 `pnpm db:cf:create`。
2. 把返回的 `database_id` 复制到 `wrangler.jsonc`。
3. 运行 `pnpm db:generate`。
4. 运行 `pnpm db:cf:migrate:local`。
5. 运行 `pnpm db:cf:migrate:remote`。

Better Auth：

1. 用 Wrangler secrets 设置 `BETTER_AUTH_SECRET`。
2. 把 `BETTER_AUTH_URL` 设置为部署后的 Worker origin。
3. 再次部署。
4. 验证注册、退出、登录和受保护 dashboard 访问。

Stripe billing：

1. 设置 `STRIPE_SECRET_KEY`、`STRIPE_WEBHOOK_SECRET`、`STRIPE_PRICE_ID`、`BILLING_SUCCESS_URL`、`BILLING_CANCEL_URL` 和 `BILLING_PORTAL_RETURN_URL`。
2. 运行 `pnpm db:generate`。
3. 运行 `pnpm db:cf:migrate:local`。
4. 运行 `pnpm db:cf:migrate:remote`。
5. 在 Stripe 中为 `/api/stripe/webhook` 配置 webhooks。

R2 storage：

1. 运行 `wrangler r2 bucket create shipstack-files`。
2. 确认 `wrangler.jsonc` 中存在 `FILES` binding。
3. 运行 `pnpm db:generate`。
4. 运行 `pnpm db:cf:migrate:local`。
5. 运行 `pnpm db:cf:migrate:remote`。

完整 checklist 见 [部署验证](../DEPLOYMENT.md)。
维护者 release flow 见 [Release Checklist](../RELEASE.md)。

## 验证情况

快速 release gate audit：

```bash
pnpm release:audit
```

在当前 workspace 里，这个 audit 会显示本地 gates 已通过，但最终 release
仍被缺少 git remote 配置和 Wrangler 未登录阻塞。

当前本地验证通过：

```bash
pnpm verify:release
```

Smoke suite 会验证生成的 base、database、auth、billing、storage、API keys、OpenAPI
和 API rate limit 应用，包括 D1 migrations、API routes、trusted-origin CORS 行为、
浏览器 auth flow、Stripe webhook 幂等性和 entitlement status、R2 文件上传/列出/删除流程、
API key creation/revocation 和 bearer auth、OpenAPI generation 和
`/api/openapi`、API rate limiting、生成应用 lint checks，以及 base 应用的 Wrangler deploy
dry-run。Release verification 也会通过 `pnpm pack:check` 检查 npm package tarball
内容。这个检查会把打包后的 tarballs 安装到临时 workspace，用 `create-shipstack`
创建应用，用打包后的 `shipstack` CLI 安装 database、auth、billing、storage、API
keys、OpenAPI 和 API rate limit modules，然后运行 `shipstack doctor`。

仓库包含一个手动触发的 `Release npm Packages` GitHub Actions workflow，用于发布
`@shipstack/core`、`@shipstack/cli` 和 `create-shipstack`，并启用 npm
provenance。正式发布前先用 dry-run 模式运行一次。

可选外部部署验证可以运行：

```bash
pnpm smoke:temporary-deploy
```

它会使用 Cloudflare temporary account flow 上传生成应用，并检查已部署的
`/health`、`/api/health` 和 `/api/v1/me` routes。它之前在当前 workspace
通过过，但 2026-06-24 最新一次尝试在上传前因 Wrangler `fetch failed` 网络错误失败。

## 已知限制

- 还没有记录真实账号下的 Cloudflare 部署验证。
- 当前 workspace 还没有确认远端 GitHub Actions 结果。
- SDK generation、teams 和 admin features 不在本 release 范围。
- MVP 路径只支持 Cloudflare-first providers：Workers、D1、R2 和 Stripe。
- OAuth provider setup 是可选项，不属于本地首次运行路径。
- CLI module system 目前刻意保持很小，只支持 `database`、`auth`、`billing`、`storage`、`api-keys`、`openapi` 和 `api-rate-limit`。
